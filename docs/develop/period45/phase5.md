# Phase 5: Gap Priority And Closeout

Status: planned.

## Priority Policy

Incomplete work is ordered by:

1. false usable-capability or observable correctness claims;
2. missing common render/compute/resource semantics;
3. production memory, binding, synchronization, query, and persistence paths;
4. advanced geometry, ray tracing breadth, and external interop;
5. specialized Metal semantics and optional performance guarantees.

Dependencies override apparent feature size. Native heaps precede sparse
residency pressure; executable resource tables precede voxel-world scale
claims; native GPU query resolution precedes profiling claims.

## Deliverables

- A grouped gap backlog with stable ledger IDs.
- Recommended implementation periods with explicit acceptance evidence.
- Updated inventory, roadmap, checklist, and backend completion routing.
- A Period 45 closeout that states what was audited, what remains incomplete,
  and why no incomplete row was promoted.

## Acceptance

- Every incomplete ledger row has one follow-up bucket.
- The next implementation period is unambiguous.
- Period 45 is marked complete only after the drift check and required API/
  backend validation commands pass.
