# Period 2

Period 2 is reserved for broader application coverage after the Period 1 polish
pass. The goal is to grow vkmtl toward practical parity with the parts of Metal
and Vulkan that real applications need, without leaking backend-specific API
shapes into normal user code.

Likely themes:

- feature and limit queries
- more texture, sampler, render pass, pipeline, transfer, and compute coverage
- multi-window or multi-surface examples
- practical application examples that combine subsystems
- portability rules for features that do not map 1:1 between Metal and Vulkan

Concrete phases should be added here once Period 1 is closed.
